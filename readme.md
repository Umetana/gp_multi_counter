GP MULTI COUNTER v1.1.0

GP MULTI COUNTER は、
OBS 配信向けのマルチカウンター管理・表示システムです。

操作・設定と表示を分離し、
操作は一貫、表示は自由に構成できることを目的としています。


────────────────────────
基本構成（必須ファイル）
────────────────────────

GP MULTI COUNTER の基本セットは、以下の 4 ファイルで構成されます。


1. display.html
────────────────
表示専用ファイルです。

・OBS の「ブラウザソース」に追加して使用します
・カウンターデータを受信し、画面に表示します
・表示するカウンターは、表示用 HTML のファイル名から自動判別されます
・カウンター1ならdisplay_counter1.html,カウンター2ならdisplay_counter2.htmlなどファイル名のcounter*で自動判別
・ベースとなる表示ファイルをコピーし、ファイル名を変更するだけで
  複数カウンターに対応できます
・コードを書き換える必要はありません


2. control_ops.html
──────────────────
配信中の操作パネルです。

・カウントの増減、リセットなどの操作を行います
・カウンターの状態を保存・配信します

3. control_setting.html
──────────────────────
設定用パネル（通常ブラウザ向け）です。

・色、フォント、サイズ、目標値など、すべての設定項目を編集できます
・見やすさ、操作性を重視した UI になっています
・注意：通常ブラウザ（Chrome 等）で行った設定は、そのままでは OBS 側に反映されません
  （ブラウザ環境が異なるため、保存領域が共有されません）
・設定を OBS 側へ移行するために、設定内容を JSON として表示・コピーできる機能があります

・設定データの表示／読込欄は、OS・ブラウザ標準のコピー＆ペースト操作を使用します
  （例：CTRL+A で全選択、CTRL+C でコピー、CTRL+V で貼り付け）
・専用のコピー／ペーストボタンはありません


4. control_setting_obs.html
──────────────────────────
OBS 内完結型の設定パネルです。

・OBS ブラウザの制約を考慮した UI になっています
・機能は control_setting.html と同等です（UI 方式のみ異なります）
・OBS 上で開いてそのまま設定・反映できます


────────────────────────
基本的な使い方（最短）
────────────────────────

【1】表示を追加する
・OBS で「ブラウザソース」を追加します
・display.html を指定します
・カウンターごとに表示用 HTML を分けて使用します

【2】操作パネルを開く
・control_ops.html を開き、カウント操作を行います

【3】設定を行う（どちらか）
A）OBS 内で設定したい場合（推奨）
  → control_setting_obs.html を OBS 上で開いて設定します

B）通常ブラウザで快適に設定したい場合
  → control_setting.html を Chrome 等で開いて設定します
  → 設定 JSON をコピーして、OBS 側へ移行します


────────────────────────
表示のカスタマイズ
────────────────────────

・表示用 HTML（display.html）は自由に改変・作成可能です
・数値表示、進捗バー、演出表示など、用途に応じて拡張できます
・表示側は共通のカウンターデータを受信して動作します

※ データ仕様の詳細は、技術者向け仕様書を参照してください


────────────────────────
設計方針
────────────────────────

・操作は人が行う
・判断や検出は自動化しない
・カウンター状態は一元管理
・表示は分離・自由


────────────────────────
動作について
────────────────────────

・本システムは BroadcastChannel API を使用しています
・サーバーや外部ツールは不要です
・OBS 環境での使用を前提としています
